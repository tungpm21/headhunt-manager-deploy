## 🟡 Warnings (Nên sửa)
1. **Rác Hệ Thống (Orphaned Files) khi thay CV mới**
   - File: `src/app/api/candidates/[id]/cv/route.ts` & `src/lib/actions.ts`
   - Nguy hiểm: Hiện tại khi User upload đè một CV mới cho ứng viên, code chỉ cập nhật DB link mới mà **chưa xóa file CV cũ** trong thư mục `public/uploads/cvs/`. Lâu ngày ổ cứng server của anh sẽ bị "phình to" vì chứa toàn file rác.
   - Cách sửa: Trong API upload, trước khi ghi file mới, nếu ứng viên đã có `cvFileUrl`, sử dụng `fs.unlink` để xóa file cũ đi.

2. **Dễ bị tấn công DDoS / File Bomb vào API Upload CV**
   - File: `src/app/api/candidates/[id]/cv/route.ts`
   - Nguy hiểm: Thiếu cơ chế **Rate Limiting** (giới hạn số lần gọi API). Một kẻ tấn công ngầm định có thể viết script gọi API này liên tục hàng ngàn lần một giây và đẩy file lên làm treo (crash) server hoặc đầy ổ cứng ngay lập tức.
   - Cách sửa: Thêm logic Rate Limiting (VD: dùng thư viện tùy chọn hoặc check cache/Redis) để giới hạn mỗi User chỉ được upload tối đa 10 file/phút.

## 🟢 Suggestions (Tùy chọn)
1. **Check N+1 Query Validation (Performance)**
   - File: `src/lib/candidates.ts`
   - Chẩn đoán: Trong hàm `getCandidates`, chúng ta dùng `CANDIDATE_TAG_INCLUDE`. Prisma đã tự tối ưu rất tốt, tuy nhiên nếu số lượng tags quá khủng khiếp, nó vẫn có thể gây chậm. Khuyến nghị giữ nguyên hiện tại, nhưng khi scale >10.000 ứng viên thì cân nhắc đổi cách lấy Tags nếu DB chậm đi.

2. **File Extension Sanitization (Security)**
   - File: `src/app/api/candidates/[id]/cv/route.ts`
   - Chẩn đoán: Mã lấy đuôi file `const ext = file.name.split(".").pop();` lấy trực tiếp extension từ User Input thay vì từ MIME Type thực tế. 
   - Cách sửa: Nên chuẩn hóa (sanitize) extension này hoặc map dựa trên `file.type` để an toàn 100%.
