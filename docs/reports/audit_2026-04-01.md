### 2. JWT secret cho employer có fallback mặc định, có thể bị giả mạo token
- File:
  - `src/lib/employer-auth.ts`
  - `src/proxy.ts`
- Dấu hiệu:
  - [`src/lib/employer-auth.ts`](/d:/MH/Headhunt_pj/src/lib/employer-auth.ts#L6) và [`src/proxy.ts`](/d:/MH/Headhunt_pj/src/proxy.ts#L8) đều dùng chuỗi mặc định `employer-jwt-secret-change-in-production` khi thiếu env.
- Nguy hiểm:
  - Chỉ cần môi trường production bị cấu hình thiếu `EMPLOYER_JWT_SECRET`, attacker có thể tự ký cookie hợp lệ và vào employer portal như tài khoản bất kỳ.
  - Đây là kiểu lỗi “không nổ lúc dev, nhưng nổ rất nặng lúc deploy”.
- Cách sửa:
  - Bắt buộc `EMPLOYER_JWT_SECRET` phải tồn tại khi boot app.
  - Xóa hoàn toàn fallback default secret.
  - Nếu env thiếu thì fail fast ngay khi khởi động.
  - Rotate secret nếu từng deploy với fallback này.

### 3. Endpoint upload CV public có thể bị spam lưu trữ và tạo file rác
- File:
  - `src/app/api/public/apply-cv/route.ts`
  - `src/lib/public-actions.ts`
- Dấu hiệu:
  - [`src/app/api/public/apply-cv/route.ts`](/d:/MH/Headhunt_pj/src/app/api/public/apply-cv/route.ts#L11) cho anonymous upload file trực tiếp.
  - Endpoint chỉ check MIME/size, không rate limit, không CAPTCHA, không ràng buộc upload phải đi kèm submit thành công.
  - [`src/lib/public-actions.ts`](/d:/MH/Headhunt_pj/src/lib/public-actions.ts#L529) chỉ xử lý tạo application sau đó, không cleanup file nếu submit lỗi/trùng.
- Hậu quả:
  - Người ngoài có thể bơm nhiều file 5MB để ăn storage/băng thông.
  - Hệ thống sẽ tích lũy file mồ côi nếu user upload xong nhưng không submit hoặc submit bị từ chối.
- Cách sửa:
  - Thêm rate limiting theo IP.
  - Chỉ cho upload qua flow submit hoàn chỉnh hoặc dùng upload token ngắn hạn.
  - Xóa file đã upload nếu `submitApplication` thất bại.
  - Cân nhắc CAPTCHA cho form public.
