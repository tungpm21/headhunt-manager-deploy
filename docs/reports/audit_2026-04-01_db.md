## 🟡 Warnings (Nên sửa)

### 1. `deleteLanguage` và `deleteExperience` không verify ownership

**File:** `src/lib/candidate-detail-actions.ts` — Lines 183–195, 273–287

**Triệu chứng:**
```typescript
// deleteLanguageAction: chỉ check id, không check candidateId
await deleteLanguage(id);

// deleteExperienceAction: chỉ check id, không check candidateId
await deleteExperience(id);
```
Hàm `deleteLanguage(id)` và `deleteExperience(id)` trong data layer chỉ xóa theo `id` mà không verify rằng language/experience đó thực sự thuộc `candidateId` từ form.

**Hậu quả:**
Một user nội bộ (MEMBER) có thể gửi request với bất kỳ `id` nào và xóa language/experience của UV bất kỳ, không chỉ UV mà họ đang xem. Trong team headhunt nhỏ thường tin nhau nên risk thực tế thấp, nhưng về nguyên tắc là lỗ hổng IDOR.

**Cách sửa:**

Trong `candidate-language.ts`:
```typescript
export async function deleteLanguage(id: number, candidateId: number) {
  return prisma.candidateLanguage.deleteMany({
    where: { id, candidateId }, // verify ownership
  });
}
```
Tương tự cho `deleteExperience` trong `work-experience.ts`. Cập nhật lại caller trong `candidate-detail-actions.ts` để truyền `candidateId` xuống.

### 2. `deleteCandidateCV`: Xóa file storage trước khi commit DB

**File:** `src/lib/candidate-cv.ts` — Lines 82–113

**Triệu chứng:**
```typescript
await deleteFile(existing.fileUrl); // ← File bị xóa
await prisma.candidateCV.delete({ where: { id: cvId } }); // ← Nếu lỗi ở đây...
```
Nếu `prisma.candidateCV.delete()` fail (DB timeout, conflict...) sau khi `deleteFile()` đã chạy, file trên Vercel Blob đã bị xóa vĩnh viễn nhưng record trong DB vẫn còn — dẫn đến broken link.

**Hậu quả:**
UV mở link CV sẽ thấy 404. Không mất data quan trọng, nhưng gây mất trải nghiệm.

**Cách sửa đơn giản:** Đảo thứ tự — xóa DB trước, storage sau:
```typescript
await prisma.candidateCV.delete({ where: { id: cvId } }); // DB commit trước
await deleteFile(existing.fileUrl).catch(console.error); // File xóa sau, fail silent OK
```

## 🟢 Suggestions (Tùy chọn)

### 3. Debounce search dùng `setTimeout` thay vì `useDebounce` hook

**File:** `src/components/candidates/candidate-filters.tsx` — Line 92

```typescript
setTimeout(() => update({ search: q || null }), 300); // timer không được cleanup
```
`setTimeout` không được cleanup khi component unmount hoặc khi user gõ nhanh. Nên dùng `useCallback` + `clearTimeout`, hoặc extract ra `useDebounce` hook.

### 4. Language filter chỉ exact match, không hỗ trợ "Tất cả tiếng Anh"

**File:** `src/lib/candidates.ts` — Line 55-61

Filter hiện tại `equals: filters.language` là exact match. Nếu user gõ "Tiếng Anh" nhưng data có "English" hoặc "tiếng anh" thì bỏ sót. Filter đã dùng `mode: "insensitive"` nhưng chỉ giải quyết case. Nên cân nhắc normalize tên ngôn ngữ khi nhập vào `CandidateLanguage`.

### 5. Retire `Candidate.cvFileUrl` + `cvFileName` về sau

**File:** `prisma/schema.prisma` — Lines 181-182

Hai trường này hiện được giữ để compat với CV viewer cũ. Sau khi migration ổn định và UI mới được QA xong, nên làm ticket retire legacy với migration xóa trường này khỏi schema để tránh confusion về "nguồn thật" của CV.
